# 广告客户开通与投放准备进度管理面板

一个用于模拟广告业务中**客户开通、账户准备、投放准备、状态跟进、异常预警和过程复盘**的进度管理面板 Demo。

> ⚠️ **数据安全声明**：本项目使用**全部模拟数据**，不包含任何真实公司、真实客户、真实用户或平台内部信息。客户名称、账号 ID、渠道经理、跟进记录等均为虚构内容，仅用于业务场景演示和作品展示。

## 🎯 项目背景

广告运营 / 投放团队日常需要在多客户、多账户、多行业、多渠道经理并行推进的场景下：

- 统一查看客户开通和投放准备状态
- 识别高优 / 高预算 / 卡住的重点客户
- 提前发现异常风险（跟进超时、素材拒审、开户失败等）
- 查看单个账号详情并沉淀跟进历史
- 复盘典型案例，形成可复用经验

本项目模拟这一完整场景，作为**招聘测试题作品 Demo**。

## 🧭 页面结构

面板包含 5 个主要 Tab：

| Tab | 功能 |
|---|---|
| 📊 数据概览 | 13 张统计卡片 + 10 张 ECharts 图表，展示客户池整体状态 |
| 📋 客户明细 | 2000+ 条客户/账户表格，支持 17 项筛选、搜索、排序、编辑、详情、导出 |
| ✅ 待办中心 | 5 个重点客户列表：高优待跟进 / 超期未更新 / 审核卡住 / 高预算未开通 / 可进入测试投放 |
| 🚨 异常预警 | 8 类风险规则自动识别，按高/中/低风险分级，展示原因和建议动作 |
| 📚 复盘沉淀 | 12+ 条模拟复盘案例，支持增删改查、案例类型筛选、账号关联 |

## 🗂 数据字段设计

每条客户/账户记录包含 33 个字段，覆盖 4 大模块：

**基础信息**：客户来源 / 账号ID / 账号名称 / 国家地区 / 行业 / 渠道经理 / 客户体量 / 预估月预算 / 客户优先级 / 投放意向

**推进状态**：当前状态 / 卡点类型 / 跟进情况 / 下一步动作 / 最近跟进时间 / 线索创建时间 / 跟进聊天截图

**投放准备**：投放平台 / 广告账户状态 / 素材状态 / 素材数量 / 素材审核结果 / 测试预算 / 投放阶段 / 首轮测试日期 / 投放准备度评分 / 投放准备状态

**元数据**：数据来源类型 / 创建人 / 创建时间 / 更新时间

## 📐 投放准备度评分逻辑

每条客户/账户按 5 个维度打分（各 20 分，总分 100）：

| 维度 | 满分 | 打分规则 |
|---|---|---|
| 基础信息完整 | 20 | 账号ID/名称/国家/行业/渠道经理均不为空 |
| 广告账户状态 | 20 | 已开户=20，开户中=10，其他=0 |
| 材料/资质状态 | 20 | 无材料问题=20，部分问题=10，严重缺失=0 |
| 素材状态 | 20 | 审核通过=20，已上传/部分通过=10，其他=0 |
| 预算明确度 | 20 | 有预算+高/中意向=20，有预算+低意向=10，无预算/无意向=0 |

**总分 → 状态映射**：
- 80-100 → 可进入测试投放
- 60-79 → 基本就绪，仍需补充
- 40-59 → 准备不足
- 0-39 → 暂不适合推进

## ⚠️ 异常预警规则

自动识别 8 类风险：

| 预警类型 | 触发条件 | 等级 |
|---|---|---|
| 跟进超时 | 最近跟进超过 7 天 | 中风险 |
| 高预算卡住 | 月预算 ≥ 5 万且未开通 | 高风险 |
| 高优客户未推进 | P0/P1 且状态为未跟进/暂停推进 | 高风险 |
| 素材审核失败 | 素材状态 = 审核拒绝 | 高风险 |
| 账户开通失败 | 账户状态 = 开户失败 | 高风险 |
| 低准备度 | 准备度评分 < 60 | 中风险 |
| 已开户但素材未准备 | 账户已开但素材=未准备/制作中 | 中风险 |
| 高意向但未触达 | 意向=高但状态=未跟进 | 低风险 |

## 🛡 数据保护规则

三种数据来源类型受不同规则约束：

| source_type | 是否可删 | 是否可编辑 |
|---|---|---|
| 表格上传 | ❌ 不可删除 | ✅ 可编辑 |
| 模拟数据 | ❌ 不可删除 | ✅ 可编辑 |
| 手动新增 | ✅ 可删除（二次确认） | ✅ 可编辑 |

**账号 ID 防重复**：任何新增/上传时，若 `pro_account_id` 已存在，将被标记为错误，提示"请勿重复添加"。

## 🛠 技术栈

- **框架**：React 18 + TypeScript 5 + Vite 5
- **UI 库**：Ant Design 6
- **图表**：Apache ECharts 6 (echarts-for-react)
- **数据层**：LocalStorage（无后端，纯前端 Demo）
- **构建部署**：GitHub Actions → GitHub Pages
- **Excel 处理**：SheetJS (xlsx)

## 🤖 AI 跟进建议

账户详情抽屉包含「AI 跟进建议」模块，当前基于规则模板自动生成 80-150 字的跟进建议，会根据客户优先级、预算、状态、卡点、素材状态、账户状态、准备度评分等字段综合输出。

**说明**：当前建议由规则生成，用于模拟 AI 辅助跟进场景；后续可接入大模型 API，根据完整的历史跟进记录生成更完整的处理建议。

## 🌉 岗位迁移说明

本项目原型聚焦广告业务投放前链路的过程管理。虽然业务场景来自跨境广告全行业客户开通，但其中的**账户准备、素材审核、预算确认、异常预警和进度追踪**，与游戏广告投放岗位中的前置准备和执行管理具有共通性。

若迁移到游戏广告投放场景，可在保留当前主线的基础上，进一步扩展：
- 游戏品类与买量目标标签（IAA / IAP / 混合变现）
- 素材创意方向沉淀（题材、卖点、爆款规律）
- 投放指标看板（ROI / LTV / 回本周期）
- 复盘分析模块（人群包、跑量素材、优化动作）

这些方向作为可拓展方向，不纳入当前主功能，以保持项目主线清晰。
